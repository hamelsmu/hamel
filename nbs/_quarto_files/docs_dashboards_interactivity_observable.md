# Dashboards with Observable JS

## Introduction

Quarto includes native support for [Observable
JS](https://observablehq.com/@observablehq/observables-not-javascript),
a set of enhancements to vanilla JavaScript created by [Mike
Bostock](https://en.wikipedia.org/wiki/Mike_Bostock) (also the author of
[D3](https://d3js.org/)). Observable JS is distinguished by its
[reactive runtime](https://github.com/observablehq/runtime), which is
especially well suited for interactive data exploration and analysis.

The creators of Observable JS (Observable, Inc.) run a hosted service at
<https://observablehq.com/> where you can create and publish notebooks.
Additionally, you can use Observable JS (“OJS”) in standalone documents
and websites via its [core libraries](https://github.com/observablehq).
Quarto uses these libraries along with a
[compiler](https://github.com/asg017/unofficial-observablehq-compiler/tree/beta)
that is run at render time to enable the use of OJS within Quarto
documents.

## Walkthrough

Quarto Dashboards are a great way to present interactive OJS
visualizations. Below we’ll provide a complete example which will give
you a high level view of the basics. If you want to learn more, please
see the complete documentation on [Using OJS with
Quarto](../../../docs/interactive/ojs/index.html).

This example covers many of the techniques you’ll use when creating
dashboards with OJS, including reactive calculations and more advanced
layout constructs like sidebars and tabsets. Here is the interactive
document we’ll be building:

<a href="../images/penguins-ojs.png" class="lightbox"
data-gallery="quarto-lightbox-gallery-1"><img
src="../images/penguins-ojs.png" class="border img-fluid"
alt="A screenshot of a Palmer Penguins dashboard. The navigation bar shows two pages: Plot (active) and Data. On the left a sidebar with an image of penguins and two inputs: a range input for Bill length; and a set of checkboxes for Islands. On the right a plot with histgrams of body mass facetted by sex and species, with bars colored by species." /></a>

The source code for this dashboard is below. Note that we add the  
`//| output: false` option to the first cell: this is to designate the
cell as having only intermediate computations (so it should not be
turned into a card in the dashboard layout).

Click on the numbers on the far right for additional explanation of
syntax and mechanics)

    ---
    title: "Palmer Penguins"
    author: "Cobblepot Analytics"
    format: dashboard
    ---

    ```{ojs}
    1//| output: false

    2data = FileAttachment("penguins.csv")
      .csv({ typed: true })

    3filtered = data.filter(function(penguin) {
      return bill_length_min < penguin.bill_length_mm &&
             islands.includes(penguin.island);
    })
    ```

    4# {.sidebar}

    ![](images/penguins.png){width="80%"}

    5```{ojs}
    viewof bill_length_min = Inputs.range(
      [32, 50], 
      {value: 35, step: 1, label: "Bill length (min):"}
    )
    viewof islands = Inputs.checkbox(
      ["Torgersen", "Biscoe", "Dream"], 
      { value: ["Torgersen", "Biscoe", "Dream"], 
        label: "Islands:"
      }
    )
    ```

    6# Plot

    7```{ojs}
    Plot.rectY(filtered, 
      Plot.binX(
        {y: "count"}, 
        {x: "body_mass_g", fill: "species", thresholds: 20}
      ))
      .plot({
        facet: {
          data: filtered,
          x: "sex",
          y: "species",
          marginRight: 80
        },
        marks: [
          Plot.frame(),
        ]
      }
    )
    ```

    # Data

    8```{ojs}
    Inputs.table(filtered)
    ```

<span code-cell="annotated-cell-1" code-lines="8" code-annotation="1">We
set `output: false` to indicate that this cell includes only
intermediate calculations and should not have its contents
printed.</span>

2  
<span code-cell="annotated-cell-1" code-lines="10,11"
code-annotation="2">We read the raw penguins dataset from a CSV when the
page loads.</span>

3  
<span code-cell="annotated-cell-1" code-lines="13,14,15,16"
code-annotation="3">`filtered` is a value that is automatically
recomputed when variables declared with `viewof` change (in this case

4  
<span code-cell="annotated-cell-1" code-lines="19"
code-annotation="4">Create global sidebars by adding the `.sidebar`
class to a level 1 header. Sidebars can include code cells as well as
images, narrative, and links.</span>

5  
<span code-cell="annotated-cell-1" code-lines="23,34"
code-annotation="5">Here we define our inputs using `viewof` so that the
`filtered` dataset is automatically recomputed when they change.</span>

6  
<span code-cell="annotated-cell-1" code-lines="36"
code-annotation="6">Level 1 headings (here `# Plots` and `# Data`)
create pages within the dashboard.</span>

7  
<span code-cell="annotated-cell-1" code-lines="38,56"

`filtered` dataset changes.</span>

8  
<span code-cell="annotated-cell-1" code-lines="60,62"
